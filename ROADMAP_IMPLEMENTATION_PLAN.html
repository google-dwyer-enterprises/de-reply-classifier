Roadmap Implementation Plan

How each of the six quality gap-fixes gets built into the fully-automated pipeline —
feasibility, approach, cost, and the test each must pass before going live · June 10, 2026

1Are these doable with zero manual effort? Yes — all six

The pipeline runs unattended on the deployed worker (scrape → filters → website verification → review queue). Every fix below runs inside that loop with no human step. Two honest distinctions up front:

  • Items 1–4 are extensions of code that already runs on every batch — most reuse website fetches and AI calls the pipeline already pays for.
  • Item 5 is the one real build: today's 312-website audit was a one-off supervised run, not pipeline code. Making it a scheduled, unattended job means porting it — fully doable since every building block already exists, but it's genuine work, not a switch to flip.
  • Item 6 needs no new human behavior at all — it harvests the approve/reject clicks the reviewer already makes as its training labels.

The one rule that governs every new gate: uncertain never auto-rejects. Each check resolves to accept, reject with quoted evidence, or unknown → review queue. A wrongly rejected lead is scraping money burned (~$0.40/lead), so rejection always requires hard evidence or high confidence — the same asymmetric principle the reseller layers were measured on.

2The six fixes — how each one works

1 — US/Canada gate

Closes: 12 foreign-HQ leads (Ksubi/AU, Manukora/NZ, Wild/UK, BAGJUMP/Austria…)

Why it's automatable: three independent signals already exist, no judgment needed — and the audit proved each works:

  1. Scraped country fields — already stored with every lead. Caveat the audit exposed: occasionally wrong (one company said "Germany", is actually NZ) — so a signal, not a verdict.
  2. The homepage the system already fetches — currency symbols (£, €, AUD), footer addresses, .com.au/.co.uk links. Zero extra cost; the page is already in hand.
  3. The web search the system already runs for hard cases — "where is X headquartered" is exactly what it surfaces.

Build: a deterministic pre-check at scrape time (all country fields agree on foreign → reject with evidence), geo-signals added to the existing site read, and an HQ-country answer added to the AI's output. Foreign + high confidence → reject; foreign + unsure → flagged "possibly foreign" for review.

Effort: smallCost: ~$0–5/mo
Go-live test: must flag ≥10 of the 12 known foreign leads and 0 of the 478 verified passes.

2 — Ownership & true-size check

Closes: 14 corporate-owned/oversized leads (Briogeo→Wella, PrettyLitter→Mars, Pura Vida ~1,000 employees)

Why it's automatable: the audit found every one of these with a single web search ("<company> acquired parent company employees"). The pipeline already makes identical search calls for reseller confirmation — this is one more question of the same shape. Scraped size data demonstrably lies (Pura Vida was listed as small), so the search is the only reliable source.

Build: one search per new company (results cached forever — a company is never re-paid for), returning parent company, independence, and a size estimate with evidence. Verdict policy: enterprise-size + high confidence → reject; subsidiary of a major parent → review queue by default, because "acquired but operating independently" (Back to Nature/Barilla) is a judgment call — this is the one item needing a single policy decision from Victor, encoded once as config.

Effort: small–mediumCost: ~$60–100/mo (the one new per-company search)
Go-live test: ≥12 of the 14 known corporate/oversized leads flagged; 0 false positives — sub-brand owners like Vetnique must not trip it.

3 — MLM detector

Closes: 7 MLM leads (Seacret, Stella & Dot, L'BRI, BELLAME)

Why it's automatable: MLMs self-identify loudly — "Join", "Become a Consultant", "Host Rewards", income-disclosure pages. The audit caught every one from its own website.

Build: two cheap layers — a vocabulary check on the scraped description/keywords (flag only, never reject on a description alone; descriptions lie), then the existing site read answers "is this an MLM?" from the nav/footer links it already extracts. MLM + high confidence → reject; unsure → review.

The known trap: influencer/"brand ambassador" programs are not MLMs — the check must require sales-downline structure, not just the word "ambassador" (Pura Vida-style programs must pass).

Effort: smallCost: ~$0
Go-live test: 4/4 known MLM companies flagged, 0 false positives on the 478.

4 — Widened site check: category + name-match + real store

Closes: 6 not-actually-DTC leads (services, dealer-only catalogs) + Victor's "all three things match" rule

Why it's nearly free: the system already fetches each undecided company's homepage and pays for one AI judgment on it. Widening what that judgment answers costs essentially nothing extra. The same read will now also verify:

  • Category against the approved/banned/out-of-scope lists — from the live site, not scraped data
  • Name consistency — site branding vs company name vs domain (mismatch → review, never auto-reject: a mismatch can be innocent, like Endangered Species Chocolate living at chocolatebar.com)
  • A real online store exists — cart/checkout present vs dealer-locator or service-booking pages (catches Marge Carson-style "we don't sell direct" sites)

Build: one prompt upgrade (versioned, regression-tested), shipped together with items 1 and 3 so it's a single change and a single test run. One addition recommended: also run this read for companies the free catalog-check already confirmed, so the whole pool gets category/name/store checks (~$15–20/month extra).

Effort: smallCost: ~$0–20/mo
Go-live test: the 6 known not-DTC leads flagged; chocolatebar.com lands in review, not rejected; 0 new false rejections.

5 — Scheduled full-pool audit the one real build

Closes: drift — sites that pivot, brands that get acquired after acceptance, and anything novel that slips every gate

The honest caveat: today's 312-website audit ran as a one-off supervised multi-agent session — that exact mechanism is not a deployable scheduled job. Making it zero-touch means porting it into pipeline code. Fully doable: every building block (polite site fetching, signal extraction, the widened AI judgment, web-search fallback) already exists in the verification module — the port is essentially "re-run the widened checks over every accepted company, compare to the stored verdicts, report what changed."

Build: an audit module that re-judges accepted companies in batches (bypassing the verdict cache), diffs new verdicts against stamped ones, emails a change report through the existing notifier, and drops changed companies into the review queue. Scheduled exactly like the existing nightly refresh job — a second cron service on the same infrastructure. Cache-aging rides along: re-judge any verdict older than ~6 months.

Cost control: monthly full pool ≈ $180–200, or a weekly 25% sample ≈ $50/month — sampling is statistically fine for drift detection.

Effort: medium (~2 sessions)Cost: $30–200 per run, by sampling
Go-live test: one scheduled run completes end-to-end unattended and produces a change report with zero human input.

6 — Learn from review decisions

Closes: no continuous measurement — today accuracy is only known via one-off audits

Why this needs zero new manual effort: the labels already exist as a by-product of normal work. Every approve/reject click in the review page records a human verdict right next to the machine's verdict. Nobody labels anything extra — the work the reviewer already does IS the training signal.

Build: when a batch fully closes (a status transition that already happens automatically), compute the agreement table: machine-passed & human-approved, review-flagged & human-rejected (the queue working as designed), and the number that matters most — escapes, leads the machine passed but the human rejected. Store per batch, add one accuracy line to the batch email that already goes out, optionally expose a quality-trend view to the client.

What stays human (by design): acting on the trend. When escapes cluster — say three foreign-HQ escapes in a month — that's the signal to tune a threshold, with the regression suite proving the tune before it ships. The measuring is automatic; the steering stays deliberate.

Effort: mediumCost: ~$0
Go-live test: after the next two production batches, per-layer accuracy appears in the batch email — with nobody having done anything new.
