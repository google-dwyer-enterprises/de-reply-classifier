Category vs domain pilot

One-off comparison test: does asking Prospeo for "owners in these e-commerce industries" beat asking for "owners at these specific domains" — on both lead quality and total reach? Script: scripts/prospeo_category_pilot.py. Read-only — writes an XLSX to exports/, no DB writes.

Headline result

Pass A — Domain mode (today)Pass B — Category mode (new)
Brand rate 5 / 10 = 50% 9 / 10 = 90%
Addressable pool 52k hand-curated domains ~123,700 owner-level people across 9 e-com industries
5,000-leads/week target Out of reach Comfortably in range
Direction: Category mode produced cleaner brand leads (fewer agencies/resellers slipping through) and opens a much larger addressable pool than our curated domain list. The signal is strong on a 10-lead-per-side sample but not conclusive — a 50-lead follow-up run is the right gate before switching production.

How it works

inclusion_clean.csv (52k domains) │ │ load_sample_domains() — reservoir sample, 60 random domains ▼ sample domains │ │ ┌─────────────────────────────────────┐ │ │ │ ▼ ▼ │ PASS A — Domain mode PASS B — Category mode │ │ │ │ │ POST /search-person │ POST /search-person │ filters.company.websites │ filters.company_industry │ .include = [<15 domains>] │ .include = [9 industries] │ filters.person_job_title │ filters.person_job_title │ .include = PILOT_TITLES │ .include = PILOT_TITLES │ (batches of 15) │ (paginate until 10 results) │ │ ▼ ▼ └─────────────────┬───────────────────┘ │ POST /enrich-person (only_verified_email: true) │ 1 credit per verified email, free otherwise ▼ enriched leads │ │ rule filter (agency tokens + marketplace domains) │ LLM grey-zone (Haiku, prompts/agency_filter.txt) ▼ classified leads → exports/category_pilot_<ts>.xlsx (Pass A sheet · Pass B sheet · Summary)

Both passes share the same downstream filter — only the upstream "what to ask Prospeo for" changes. That's the variable being tested.

What the pilot tests

Pass A — DomainPass B — Category
Question to Prospeo "Find owners at these 60 specific websites." "Find owners at any company in these 9 industries."
Prospeo filter shape company.websites.include (nested — empirically the only working domain shape) company_industry.include (singular, top-level — only working industry shape, per FINDINGS.html §6)
Title filter PILOT_TITLES = owner-only set: CEO, Chief Executive Officer, Founder, Owner, Co-Owner, President, Founder and CEO, Managing Director, Chairman
Downstream filter Rule pass (agency tokens + marketplaces) → Haiku grey-zone classifier → brand / agency / reseller / marketplace / unknown
Target sample 10 enriched results per pass (apples-to-apples)

CLI

# Dry-run: shows the filters that would be sent, no API calls, no spend
python scripts/prospeo_category_pilot.py --dry-run

# Live run, capped to ~$0.40
python scripts/prospeo_category_pilot.py --max-credits 20

# Larger live run with confirmation skipped (CI / automated)
python scripts/prospeo_category_pilot.py --max-credits 40 --yes

# Rule-only filter (skip the Haiku grey-zone classifier)
python scripts/prospeo_category_pilot.py --max-credits 20 --skip-llm

# Try Pass B with a US country filter (off by default — Prospeo may reject)
python scripts/prospeo_category_pilot.py --max-credits 40 --with-country

Flags:

  • --max-credits N — hard budget cap. Aborts before any call that would push spend past N. Default 40 (~$0.80).
  • --dry-run — print the JSON filters for both passes and exit. No API calls.
  • --target-results N — leads per pass (default 60). The pilot used 10.
  • --skip-llm — rule-only agency filter (no Haiku calls).
  • --yes — skip the y/N confirmation prompt before live spend.
  • --with-country — add company_location_search.include = ["United States"] to Pass B. Off by default — raw-string location filter is unverified and may return INVALID_FILTERS.

What jumps out:
  • Pass B leads look like recognizable DTC brands — apparel, beauty, food, baby goods, golf accessories. Exactly the profile Atlas signs.
  • Pass A pulled the same reseller (Atlanta Cutlery) three times — three distinct Presidents at one multi-brand reseller. When a curated domain is itself a reseller, the bot collects more wrong people, not fewer.
  • Pass A also burned 2 of 10 slots on "no verified email." Pass B had zero such skips.

Caveats & things to watch

  • Small sample. 10 leads per side. The 90% / 50% gap is large enough to be directionally meaningful, but a 50-lead follow-up is the right gate before switching production over. Expect Pass B to settle in the 70–90% range at scale, Pass A in the 40–60% range.
  • Pass B is currently global, not US-only. The first page included some non-US brands. company_location_search with raw country strings is undocumented and may return INVALID_FILTERS — gated behind --with-country until verified separately. If we need US-only, this is a small follow-up.
  • Industry strings are post-2023 LinkedIn taxonomy. Adding a new industry to PILOT_INDUSTRIES requires running it through scripts/verify_prospeo_shape.py first — the API silently accepts wrong shapes and just ignores them (same class of bug as the 15-domain websites cap).
  • Industry filter shape is company_industry, singular, top-level. Nested company.industry / company.industries shapes parse but are silently ignored, returning the no-industry baseline. The pilot script's _build_category_filter uses the verified shape.
  • The pilot does not write to the DB. Output is XLSX only. Once we commit to category mode, prospeo_sync.py will need a category-mode branch and the same per-batch persistence guarantees as the domain path.
