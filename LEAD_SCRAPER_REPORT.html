3Prospeo filter shape — what works

Industry filter is the part most likely to fail silently. API-verified across 4 separate probe runs in this engagement.

ShapeBehavior
filters.company_industry.include = [...] (singular, top-level)Works — credits charged, real results, narrows total_count
filters.company.industries.include (nested, plural)Silently ignored — 200 OK but total_count matches no-filter baseline
filters.company.industry.include (nested, singular)Silently ignored
filters.industries.include / filters.industry.includeINVALID_FILTERS (400, free)
filters.company_industries.include (plural, top-level)INVALID_FILTERS

Silent no-op trap

The wrong-but-not-rejected shapes return HTTP 200 with no error. total_count is the only signal: a filter that returns the unfiltered baseline (tens of millions) is doing nothing. Same trap exists at the value level — see §4.

What this run resolves

  • Country filter works with raw strings. {"company_location_search": {"include": ["United States", "Canada"]}} returns total_count=51,650 — a real narrow filter. No Suggestions-API resolution needed for these two values.
  • Category mode brand rate: 70.3% (n=37). Within the predicted [70–90%] range, at the low end. ~3.3× domain mode at this scale.
  • Domain mode is not viable as a primary path. 50% no-email skip rate at n=14 means half the curated domains don't return contactable owners; brand rate of those that do is only 21.4%.

9Recommended next steps

  1. ✅ Done 2026-05-15: 51-lead pilot with the expanded 12-industry / 17-title list, US+Canada filter. Brand rate landed at 70.3% — within the predicted [70–90%] range. Production-ready.
  2. If the 50-lead run holds: switch the weekly scraper to category mode as primary, keep the domain list as fallback / sanity check. Should hit 5,000/week inside the current $99 Prospeo plan.
  3. Standalone electronics pilot. Send only Appliances, Electrical, and Electronics Manufacturing + the 17 titles, 10 leads, ~$0.20. If 7+ are DTC consumer-electronics brands (Anker-like), add to PILOT_INDUSTRIES. If industrial-dominant, skip.
  4. Investigate the (no title) bucket. 17.7% positive rate at lead level is the highest-converting tier and currently has no targeting story. Likely Apollo enrichment misses on small operators. Separate workstream.
  5. Optional A/B on title tiers. Once category mode is in production, periodically pull conversion by tier-1 vs tier-2 leads to confirm the marketing-tier addition is still positive-EV. If tier-2 conversion drops below ~5%, revert to owner-only.

10Caveats & open questions

  • Pool exhaustion at 5K/week. US+Canada pool is 51,650 owner+marketing-level people across all 12 industries. At 5K leads/week with a 70% brand rate, you'd burn through unique leads in ~7 weeks. Plan for either (a) widening to UK/AU/EU later, (b) refreshing the pool as Prospeo's data updates, or (c) accepting lower weekly volume after ~month two.
  • Country filter is now verified ({"company_location_search": {"include": ["United States", "Canada"]}} returned total_count=51,650, 2026-05-15). The earlier Suggestions-API caveat no longer applies — raw country strings work for these two values. Other countries unverified.
  • Silent no-op risk on new additions still partial. Retail Groceries, Alternative Medicine, and Retail Health and Personal Care Products are in the verified-narrowing set but the second pilot's 37 leads don't cleanly tag back to a single industry. A per-industry breakdown isn't possible from the combined-OR query; trust them tentatively and watch the 50-lead next-pilot output for any industry that produces consistently low-quality leads.
  • 15-domain websites cap is enforced (PROSPEO_BATCH_DOMAINS = 15). Don't raise — verified to silently ignore entries above 15 after a 506-credit production burn.
  • Pilot does not write to DB. Output is XLSX only. Once committed to category mode, prospeo_sync.py needs a category-mode branch with the same per-batch persistence guarantees as the domain path.
  • Marketing-tier titles are a recent addition. The 8.6% lead-level conversion is historical reply data, not new-cohort behavior. Watch the first 50-lead run for drift.
